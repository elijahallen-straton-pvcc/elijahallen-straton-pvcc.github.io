    All About GitHub

    Questions and Answers

    What is a version control system?

    • Version Control system (VCS) is software that tracks and manages changes made to code and other files.

    What is Git?

    • Git is a distributed VCS that tracks changes in any set of computer files and is used for coordinating work among programmers.

    What is GitHub?

    • GitHub is an online software development platform. It is used for storing, tracking, and coloborating on projects for differnt developers.

    What is the differnece between Git and GitHub?

    • Git is a VCS while Github is a cloud based hosting service.

    Who started GitHub and how was it started?

    • GitHub was founded by Tom Preston-Werner, Scott Chacon, Chris Wanstrath, and P.J. Hyett in 2008 as a bootstrapped start-up business.

    What company owns it now?

    • GitHub was purchased by microsoft in 2018 and the company still owns it to this day.

    How much does a GitHub account cost?

    • GitHub is free to use, but they do offer paid plans for storing and collaborating on code.

    What is the Octocat?

    • Octocat is the mascot for Github which is an anthropomorphized cross between a cat and an octopuss having five octopus like arms and the head of a cat. This character has become synonymous with GitHub. Many of GitHub's users have also made artwork of this character over the years. If you ever get a 404 screen from GitHub, the mascot will appear there aswell.

    GitHub Vocabulary
    Definitions

    Repository:

    Reposistory contains all of the usesrs files and file version history.

    Commit:

    To "commit" a file is to add it into your Repository.

    Fork:

    A Fork is a new Repository that shares code and visibilty settings with the original "upstream" Repository.

    Push:

    A Push is a command that updates the remote branch with local Commits.

    Pull Requests:

    Pull Requests let you tell others about a change you may have pushed to a branch within a repository on GitHub.

    Workflows:

    A configurable automated process that is able to run one or more jobs.

    Issues:

    Issues allow you to track your work on GitHub.

    Raw Button:

    The Raw Button allows you to view the raw content of a fie without any of its styling.

    Blame Button:

    This command allows you to examine a file line by line and see when each lone was modified and by who.
