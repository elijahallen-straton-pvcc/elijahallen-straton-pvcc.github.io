Software Testing and Verification

Kinds of Tests
    Acceptance Testing: A test done to verify if the entire system works as intended
    Integration Testing: A test done to make sure software components or functions are operating together
    Unit Testing: A test done to validate the expected performance of each software unit
    Functional Testing: Testing done by emulating buisness scenarios based off of the functional requirements often done using black box testing to verify functions
    Performance Testing: Testing how the software would perform based on differnt workloads
    Regression Testing: This test checks if newly added features break or degrade functionality
    Stress Testing: These are tests done to see the stressload a system can take before it fails
    Usabiloity Testing: A test done in order to figure out how well customers can use a system or web application to complete a task
Software Testing is Done to Detect
    Architectural Flaws
    Poor Design Decisions
    Invalid or Incorrect Functionality
    Security Vulnerabilities
    Scalability Issues
Tezting Best Practices
    Continous Testing: This is a testig technique in which developers and operations teams will test the software over the course of its development rather than after product is fully built in an effort to catch bugs early on and have less of them by the time of release.
    Configuration Management: These are management teams built to test assets and track software builds to test. Often these teams will gain access to code, requirements, design documents, models, test scripts, and results in order to help build good systems that meet requirements with minimal administrative efforts.
    Servic Virtualization: A simulated environment of the service systems that are missing or not yet completed which can enable teams to reduce dependancy and test sooner
    Defect or "Bug Tracking": Automated tools which can help development by tracking defects and measuring their scope and impact, and also uncover related issues
    Metrics and Reporting: Techniques involving reporting and analytics in order to allow for team members to share status, goals, and test results
